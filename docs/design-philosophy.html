把读轻小说
做成一件爽的事

一个以「和纸质感 + 日系美学 + 防剧透 AI」为内核的沉浸式阅读器。

Kreader 不是又一个 Calibre。它假设你的读者是在深夜、宽屏、想被一本日文轻小说带走的人—— 所以每一处设计都服务于「沉浸」与「无痛续读」：纸感双页书卷、原生振假名排版、 点词即查、术语表记忆，以及一整套只基于「你已读到哪里」的 AI 阅读助手。 下面这份文档讲清楚它为什么长这样，以及它在工程上如何落地为一套可前后端分离、可自由切换 AI 模型（QWEN / DeepSeek）的系统。

沉浸式纸感 防剧透优先 日文排版一等公民 EPUB / TXT 导入 模型可插拔 · QWEN / DeepSeek Next.js · 前后端可分离
01

缘起 · 为谁而做

机翻轻小说最痛的从来不是「翻不出来」，而是体验割裂：术语全书前后不一、断章后忘了前情、想查个词要切到字典 App、读到一半被搜索引擎剧透。Kreader 把这些痛点逐一拆掉。

PAIN 01

术语漂移

同一个「アスナ」这一章译亚丝娜、下一章译阿斯娜。Kreader 用 AI 术语表锁定角色名 / 地名 / 必杀技，全书一致。

PAIN 02

断章失忆

搁置两周再打开，不知道读到哪了。前情提要基于你已读的页数生成「上回说到…」，绝不带出未读剧情。

PAIN 03

查词出戏

点任意日语词即弹词卡：假名、罗马音、JLPT 等级、本书出现次数，一键进生词本——不离开书页。

02

四条设计原则

这四条是 Kreader 在任何取舍时刻的判据。当「好看」和「好用」冲突，按它们排序。

沈 PRINCIPLE 一

沉浸高于功能堆叠

默认界面是一本摊开的书，不是一个控制台。和纸底色、明朝体、书脊阴影、翻页动画——技术感让位于纸感。所有工具栏都可隐去，只剩书与你。

防 PRINCIPLE 二

防剧透是底线，不是选项

人物图谱、世界观百科、问答、情绪曲线——一切 AI 输出只能引用你已读到的位置。未读内容在 UI 上以「？？？」「🔒 读至第 N 章解锁」呈现。问「他最后怎么了」会被礼貌拦截。

和 PRINCIPLE 三

日文排版是一等公民

原生 <ruby> 振假名渲染、禁则处理、未来的竖排右→左。中日双语对照以段落级交错呈现，学日语的人点一段才展开译文。

渐 PRINCIPLE 四

随进度生长

图谱节点、百科词条、情绪曲线都随阅读进度「长出来」。已读 5% 时助手只懂 5%。这既是防剧透，也让陪伴感真实——它和你一起读。

03

从 Scen-Mes 工程控制台美学，转向和纸 × 明朝体 × 朱红的日系沉浸风。三套主题覆盖昼 / 夜 / 护眼三种阅读情境。

MOTIF

纸感的细节

书页有三层堆叠投影模拟厚度；中缝一道渐变阴影还原书脊；翻页用 cubic-bezier(.22,.61,.36,1) 的缓动。「氛围模式」会按章节情绪渲染星夜微光与环境音——这是通往 Fable 式沉浸上限的入口。

04

从「读得了」到「读得爽」，四层逐级递进。基础层必须扎实，上限层负责惊艳。

05

技术架构

一套 Next.js（App Router） 应用，前后端可分离：UI 是 React Server / Client Components，所有重活（解析、AI、存储）收敛到 Route Handlers，因此换前端、换部署形态都不动核心逻辑。

① 前端 / Reader

书库 · 阅读视图
RSC 渲染书架；阅读器为 Client Component，承载翻页 / 查词 / 助手面板。
主题 & 排版引擎
CSS 变量驱动三主题；<ruby> 振假名、竖排预留。
本地状态
阅读进度、生词本、设置先落 IndexedDB，可选云同步。

② 服务端 / API

/api/books · import
上传 EPUB/TXT → 解析 → 入库；支持批量导入一组文件。
EPUB / TXT Pipeline
epub.js 解目录与章节，TXT 正则智能分章（「第〇章」「序章」）。
/api/ai/* · 流式
翻译 / 问答 / 提要统一走 Provider 抽象层，SSE 流式返回。

③ 数据 & 模型

Book / Chapter / Para
SQLite / Postgres + Prisma；段落级粒度，便于定位「已读到哪」。
Glossary · 术语表
每本书一张术语表，翻译时注入 prompt 保证全书一致。
AI Provider 抽象
QWEN / DeepSeek / 自定义 OpenAI 兼容端点，可热切换。

导入流程一条线：

// app/api/books/import/route.ts —— 单本 / 批量皆走此入口 export async function POST(req: Request) { const files = await req.formData() // 一个或多个 epub/txt const books = [] for (const file of files.getAll('files')) { const parsed = file.name.endsWith('.epub') ? await parseEpub(file) // 目录树 · 封面 · 章节 · 元数据 : await parseTxt(file) // 智能分章 + 语言探测 books.push(await db.book.create({ data: parsed })) } return Response.json({ books }) // 也支持「新建空书」手动粘贴正文 }
06

AI 接入 · 模型可插拔

客户在中国大陆，首选 通义千问 QWEN 与 DeepSeek。两者均提供 OpenAI 兼容接口，因此用一层统一 Provider 抽象即可在设置里自由切换、各功能独立配置（翻译用一个模型、问答用另一个）。
